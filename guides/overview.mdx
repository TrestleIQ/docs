### Base URL

All API requests should be made to:

```
https://api.trestleiq.com
```

### Authentication

Trestle controls access to the API and data via an API key. The API key is the primary data authentication method for your account. Your usage is recorded and reported via the API key. You may use a single API key for multiple Trestle APIs.

# Trestle Product Descriptions

Here's a brief overview of Trestle's APIs:

### Validation and Verification APIs

- [Phone Validation API](https://trestleiq.com/phone-validation-api/): Validates an input phone number and returns real-time metadata signals like carrier, line type, prepaid status, and phone activity. TCPA litigator checks available as an add-on.
- [Real Contact API](https://trestleiq.com/real-contact-api/): Verify the contactability and consistency of an identity profile across name, email, phone number, physical address, and IP address. Returns powerful metadata, name match results, and contact grades. Available with add-ons for email deliverability, email age, and TCPA litigator checks.

### Enrichment APIs

- [Reverse Phone API](https://trestleiq.com/reverse-phone-api/): Trestle’s most robust enrichment API. Validates an input phone, provides relevant metadata, and returns **all potential phone owners** along with each owner’s linked addresses, emails, and alternate phone numbers.
- [Caller Identification API](https://trestleiq.com/caller-identification-api/): Designed for use cases where only the top associated identity is needed, the Caller Identification API validates the phone, provides real-time metadata, and returns **only the** **highest ranked phone owner** along with their current address, emails, and alternate phone numbers.
- [Smart CNAM API](https://trestleiq.com/smart-cnam-api/): Designed for use cases where only name data is needed, the Smart CNAM API validates the phone number and returns the **top associated name**.
- [Reverse Address API](https://trestleiq.com/reverse-address-api/): Validates an input address, returns relevant metadata, and returns all identities currently linked to the address with demographic data, phone numbers, emails, associated people, and historical addresses for each linked identity.
- [Phone Feedback API](https://trestleiq.com/phone-feedback-api/): Offers a method of providing feedback to Trestle about dialed numbers for their connected/disconnected status and confirmation of right party contact.

# FAQ

### **_How do I get an API Key?_**

[Sign up for a Developer Portal account](https://portal.trestleiq.com/signup). Once verified, you can request access to any of Trestle’s Enrichment APIs or start using Real Contact API or Phone Validation API immediately.

### **_Do you have a Postman Collection?_**

Yes, we do, and you can find it [here](https://www.postman.com/trestleiq/workspace/trestle-identity-data-apis). You will need to fork the collection to use it and an API key, for which you can [sign up here](https://dev-portal.trestleiq.com/accounts/create).

### **_Where can I check the status of the service?_**

You can find the API status on the [Trestle Status Page](https://status.trestleiq.com/).

### **_Can I submit a request using POST method?_**

POST is only supported for our Phone Feedback API. Rest of the APIs only support via GET.
